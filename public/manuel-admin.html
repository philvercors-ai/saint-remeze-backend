Gérer les signalements

Vue d'ensemble

Le dashboard affiche en temps réel le nombre de signalements par statut, puis la liste complète avec les filtres disponibles.

Filtres disponibles

  • Recherche textuelle — filtre sur le titre, la description ou le nom du citoyen
  • Statut — affiche uniquement les signalements d'un statut donné
  • Du / Au — plage de dates sur la date de création
  • Bouton ↺ — réinitialise tous les filtres

Ouvrir un signalement

Cliquez sur le bouton Détails d'une ligne. La fiche s'ouvre et le statut passe automatiquement de ⏳ En attente à 👁️ Vue. Le citoyen reçoit une notification email automatique.

Modifier un signalement

Dans la fiche ouverte, vous pouvez modifier :

  • Statut — choisir parmi : En attente, Vue, En cours, Terminée, Rejetée
  • Assigné à — nom de l'agent ou du prestataire en charge
  • Notes de la mairie — message interne ou explication visible côté citoyen

Cliquez sur Enregistrer pour valider. Le citoyen reçoit un email automatique à chaque changement de statut.

Les statuts et leur signification

StatutSignificationEmail citoyen
⏳ En attenteSignalement reçu, non encore examinéNon
👁️ VueSignalement ouvert par l'adminOui
🔧 En coursTraitement en coursOui
✅ TerminéeProblème résoluOui
❌ RejetéeNon traitable (hors compétence, doublon…)Oui

Supprimer un signalement

La suppression est possible uniquement pour les signalements archivés depuis plus d'un an. Pour les autres, utilisez le statut ❌ Rejetée.

3

Exports PDF & CSV

Lancer un export

Les boutons 📄 PDF et 📊 CSV se trouvent en haut à droite du dashboard. L'export porte sur les données filtrées et affichées — activez les filtres avant d'exporter pour obtenir un rapport ciblé.

Export PDF

  • Page 1 — titre du rapport, filtres actifs, période, graphique camembert (statuts), graphique barres (catégories)
  • Pages suivantes — une page par signalement avec toutes les informations : citoyen, contact, description, localisation GPS cliquable, photos
⏳ La génération peut prendre plusieurs secondes Un indicateur animé apparaît en bas à droite pendant la création. Le bouton PDF est désactivé pendant ce temps.

Export CSV

Le fichier CSV contient une ligne par signalement avec les colonnes : Date, Citoyen, Email, Téléphone, Catégorie, Titre, Statut, Assigné à, Description, Notes admin. Ouvrable directement dans Excel ou LibreOffice Calc.

Filtrer avant d'exporter

Pour exporter uniquement les signalements terminés du mois d'avril :

  1. Sélectionner Terminée dans le filtre statut
  2. Saisir 01/04/2026 dans « Du » et 30/04/2026 dans « Au »
  3. Cliquer sur 📄 PDF ou 📊 CSV
4

Gestion des archives

Archivage automatique

Les signalements au statut ✅ Terminée ou ❌ Rejetée sont archivés automatiquement 1 an après leur création. Cette tâche s'exécute chaque nuit à 2h00.

Archivage manuel

Pour archiver un signalement manuellement, ouvrez sa fiche et cliquez sur Archiver. Il disparaît de la vue active et passe dans l'onglet Archives.

Consulter les archives

Cliquez sur l'onglet Archives dans le dashboard. Les filtres et exports fonctionnent de la même façon.

Désarchiver

Depuis l'onglet Archives, ouvrez la fiche et cliquez sur Désarchiver pour remettre le signalement en liste active.

Suppression définitive

La suppression n'est disponible que pour les signalements archivés depuis plus d'un an. Le bouton 🗑️ devient alors actif dans la fiche. Cette action est irréversible.

🚨 Suppression irréversible Une confirmation est demandée. Les photos hébergées sur Cloudinary sont également supprimées définitivement.
5

Changer le mot de passe administrateur

Le mot de passe admin est stocké hashé en base MongoDB. La procédure utilise le script createAdmin.js exécuté localement depuis votre Mac.

⚠️ Prérequis Node.js doit être installé sur votre Mac et le dossier du projet doit être accessible.

Étape 1 — Récupérer le MONGODB_URI

  1. Connectez-vous sur render.com
  2. Ouvrez votre service backend → onglet Environment
  3. Copiez la valeur de la variable MONGODB_URI

Étape 2 — Créer un fichier .env local

Dans le dossier backend/ du projet, créez un fichier nommé .env avec ce contenu :

MONGODB_URI=mongodb+srv://... ← coller la valeur copiée

Étape 3 — Modifier le nouveau mot de passe

Ouvrez le fichier backend/createAdmin.js et modifiez la ligne :

const ADMIN_EMAIL = 'admin@saintremeze.fr';
const ADMIN_PASS = 'votre_nouveau_mot_de_passe'; ← modifier ici

Étape 4 — Lancer le script

Ouvrez le Terminal et exécutez :

cd "/Volumes/DATA_PHOTOS/@CODAGE APPLI/saint-remeze-COMPLET-v7.2 PROD/backend"
node createAdmin.js

Résultat attendu :

✅ Connecté à MongoDB Atlas
🔄 Admin mis à jour (mot de passe réinitialisé)
   Email : admin@saintremeze.fr
   Password : votre_nouveau_mot_de_passe
✅ Le mot de passe est immédiatement actif Aucun redémarrage du serveur n'est nécessaire. Reconnectez-vous sur le dashboard avec le nouveau mot de passe.

Étape 5 — Nettoyage

🚨 Important — Sécurité Après l'opération, supprimez le fichier .env local (il contient le mot de passe MongoDB) et remettez createAdmin.js à sa valeur d'origine. Ne commitez jamais ces fichiers sur GitHub.
  1. Supprimer le fichier backend/.env
  2. Remettre ADMIN_PASS = 'admin123' dans createAdmin.js (ou laisser vide)
  3. Vérifier que .env est dans le fichier .gitignore

Variables d'environnement (Render)

Ces variables sont définies sur render.com → service backend → onglet Environment. Elles ne doivent jamais apparaître dans le code source.

VariableDescriptionObligatoire
MONGODB_URI URI de connexion MongoDB Atlas (inclut login/password) ✅ Oui
JWT_SECRET Clé secrète pour signer les tokens JWT — chaîne aléatoire longue ✅ Oui
RESEND_API_KEY Clé API Resend pour l'envoi d'emails ✅ Oui (pour les emails)
EMAIL_FROM Adresse expéditeur des emails : Saint-Remèze <no-reply@saintremeze.fr> Recommandé
FRONTEND_URL URL du frontend Vercel : https://saint-remeze-frontend.vercel.app Recommandé
CLOUDINARY_CLOUD_NAME Nom du compte Cloudinary pour l'hébergement des photos ✅ Oui (pour les photos)
CLOUDINARY_API_KEY Clé API Cloudinary ✅ Oui (pour les photos)
CLOUDINARY_API_SECRET Secret API Cloudinary ✅ Oui (pour les photos)
💡 Générer un JWT_SECRET sécurisé Dans le Terminal de votre Mac, exécutez :

node -e "console.log(require('crypto').randomBytes(64).toString('hex'))"

Copiez la valeur générée (128 caractères hexadécimaux) et collez-la dans Render.
